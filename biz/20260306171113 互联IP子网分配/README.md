# 20260306171113. 互联IP子网分配

🟡 难度：中等

## 题目描述

### 业务场景

- **问题背景**：
    - 数据中心新上架了一批核心交换机，需要在它们之间建立几十条甚至上百条点对点连接（比如 Spine-Leaf 架构中，每台 Spine 和每台 Leaf 之间都要连线）。每条连接需要分配一对互联 IP 地址，传统做法是网络工程师手工在 Excel 表格里规划，一条一条填写 IP 地址。

- **导致的问题**：

    1. **效率低下**：100 条连接需要手工填写 200 个 IP 地址，耗时且枯燥

    2. **容易出错**：手工分配容易出现 IP 冲突、跳号（漏掉某些IP导致地址不连续）、或者不小心用了网络地址/广播地址
    
    3. **难以维护**：后续新增连接时，需要翻查 Excel 找到上次用到哪个 IP，容易接错

- **解决方案**：开发一个自动分配程序，输入一个地址池（如 `172.168.1.0/24`）和连接列表，程序自动按顺序切分出 /30 子网，为每条连接分配一对 IP。这样可以：

    - 几秒钟完成上百条连接的 IP 规划

    - 保证 IP 连续、无冲突、无遗漏

### 核心问题

**输入**：
- 一个 CIDR 格式的地址池（如 `172.168.1.0/24`）
- 一个连接列表，每条连接包含源设备、源端口、目标设备、目标端口

**输出**：
- 为每条连接分配一个 /30 子网，包含：
  - 子网地址（如 `172.168.1.0/30`）
  - 源端 IP（如 `172.168.1.1/30`）
  - 目标端 IP（如 `172.168.1.2/30`）

**约束**：
- 子网必须按顺序从地址池中切分，每个 /30 占用 4 个连续地址
- 若地址池容量不足，返回错误
- 输出顺序与输入连接列表顺序一致
- pool 是合法的 CIDR 格式，前缀长度范围为 /16 ~ /28
- links 数组长度范围：1 ≤ len(links) ≤ 1000

## 示例 1：基本分配

输入：
```json
{
  "pool": "172.168.1.0/24",
  "links": [
    {
      "SourceDevice": "Switch-A",
      "SourcePort": "Eth1",
      "TargetDevice": "Switch-B",
      "TargetPort": "Eth1"
    },
    {
      "SourceDevice": "Switch-A",
      "SourcePort": "Eth2",
      "TargetDevice": "Switch-B",
      "TargetPort": "Eth2"
    }
  ]
}
```

输出：
```json
[
  {
    "Link": {
      "SourceDevice": "Switch-A",
      "SourcePort": "Eth1",
      "TargetDevice": "Switch-B",
      "TargetPort": "Eth1"
    },
    "Subnet": "172.168.1.0/30",
    "SourceIP": "172.168.1.1/30",
    "TargetIP": "172.168.1.2/30"
  },
  {
    "Link": {
      "SourceDevice": "Switch-A",
      "SourcePort": "Eth2",
      "TargetDevice": "Switch-B",
      "TargetPort": "Eth2"
    },
    "Subnet": "172.168.1.4/30",
    "SourceIP": "172.168.1.5/30",
    "TargetIP": "172.168.1.6/30"
  }
]
```

解释：
- 第 1 条连接分配 172.168.1.0/30，源端用 .1，目标端用 .2
- 第 2 条连接分配 172.168.1.4/30，源端用 .5，目标端用 .6

## 示例 2：地址池满载

输入：
```json
{
  "pool": "10.0.0.0/26",
  "links": [
    // ... 16 条连接（省略具体内容）
  ]
}
```

输出：成功分配 16 个 /30 子网

解释：
- /26 有 64 个地址，可以切分出 16 个 /30 子网（16 × 4 = 64）
- 刚好够用

## 示例 3：地址池耗尽

输入：
```json
{
  "pool": "10.0.0.0/26",
  "links": [
    // ... 17 条连接（省略具体内容）
  ]
}
```

输出：
```json
{
  "error": "地址池 10.0.0.0/26 已耗尽，最多支持 16 个 /30 子网，但需要分配 17 个"
}
```

解释：
- /26 只能切分出 16 个 /30，第 17 条连接无法分配

## 相关知识点

- CIDR 子网计算
- /30 子网的地址结构（网络地址、可用IP、广播地址）
- 地址池容量计算：一个 /N 网段可以切分多少个 /30
